Git 的 core.autocrlf 设置说明

Feb 01, 2020 • 预计阅读时间 1 分钟

众所周知，Windows 和 Unix/Linux 的操作对文本换行有不同的标准。

Windows 默认是 \r\n，而 Unix/Linux 则使用 \n。

Git 的 core.autocrlf 用来指定提交时如何处理回车和换行，用法如下：

git config --global core.autocrlf true

参数说明

true提交时转换为 \n，检出时替换为 \r\n
input提交时转换为 \n，检出时不处理
false提交和检出都不处理
